Well, both those compilers should support F2003 standard, so a 
quick-and-dirty approach could be to compile by hand and link together 
to form a dll the main .F90 and .c sources of the package, namely:

libfortranc/fortranc.F90
libfortrangis/gdal.F90
libfortrangis/shapelib.F90
libfortrangis/shapelibint.c

keeping in mind that files from the directory include/ have to be 
included; the line #include "config.h" in libfortranc/fortranc.F90 
should be commented since config.h is autogenerated by autoconf, but it 
is not strictly necessary at the moment.

After that you should be able to compile the 
libfortrangis/shapelib_test.F90 and libfortrangis/gdal_test.F90 test 
sources using the generated modulefiles and libraries and the 
precompiled gdal and shapelib libraries. To make all this work smoothly 
it is probably necessary to create a projectfile or something like that.
	Please report on any success, best regards, Davide
