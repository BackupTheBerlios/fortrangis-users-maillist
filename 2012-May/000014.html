> Well, both those compilers should support F2003 standard, so a 
> quick-and-dirty approach could be to compile by hand and link together 
> to form a dll the main .F90 and .c sources of the package, namely:
>
> libfortranc/fortranc.F90
> libfortrangis/gdal.F90
> libfortrangis/shapelib.F90
> libfortrangis/shapelibint.c
>
> keeping in mind that files from the directory include/ have to be 
> included; the line #include "config.h" in libfortranc/fortranc.F90 
> should be commented since config.h is autogenerated by autoconf, but 
> it is not strictly necessary at the moment.
>
> After that you should be able to compile the

I have managed to compile FortranGIS on windows by creating a .lib file 
+ .mod file for each libfortranc and libfortrangis and linking these to 
a project including the gdal.F90 source file. At first I received 
"unresolved externals" errors but solved this by changing all the 
subroutine names. For example I changed GDALAllRegister() to 
GDALAllRegister at 0 -> @0 being the number of arguments passed times 4.
It all works fine and will be a great help for my master thesis! Perhaps 
at some point I can help out with the project.
